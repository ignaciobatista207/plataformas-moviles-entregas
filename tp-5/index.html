Control de Versiones con Git

Git es un sistema de control de versiones distribuido, ampliamente utilizado en el desarrollo de software moderno. Fue creado para gestionar proyectos de forma rápida, eficiente y colaborativa.

🎉 Novedad: Git 2.50 fue lanzado con mejoras de rendimiento en repositorios grandes.

Comandos más populares de Git

Hacé clic en un comando para ver su descripción completa.

Inicializa un nuevo repositorio Git en el directorio actual. Este comando crea un repositorio vacío listo para trabajar.

Clona un repositorio remoto en tu máquina local. Ejemplo: git clone https://github.com/usuario/repositorio.git

Muestra el estado del repositorio: archivos modificados, nuevos archivos no rastreados, cambios preparados para el commit, etc.

Agrega un archivo o cambios específicos al área de preparación (staging area) para el commit.

Crea un commit con los archivos añadidos al staging, con un mensaje descriptivo.

Sube los cambios locales al repositorio remoto. Se usa después de un commit para actualizar el repo remoto.

Obtiene y combina los cambios del repositorio remoto al local. Equivalente a git fetch + git merge.

Cambia a una rama específica. Si no existe, se puede crear con git checkout -b [rama].

💡 Tips Avanzados de Git
🌿 Trabajar con ramas

Siempre trabajá en una rama separada para cada feature o fix.

git checkout -b feature/mi-feature
🔍 Ver historial

Usá git log --oneline --graph para ver el historial de forma visual.

🧹 Limpiar cambios

git stash guarda cambios temporalmente sin hacer commit.

✏️ Modificar último commit

git commit --amend te permite editar el mensaje del último commit.
